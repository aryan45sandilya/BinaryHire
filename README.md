## тЬж Overview

**BinaryHire** is a complete Applicant Tracking System (ATS) built as a single-page React application backed by an Express REST API. It covers the entire hiring workflow тАФ from sourcing candidates and managing job specs to AI-powered screening and scheduling interviews тАФ all wrapped in a distinctive physical stationery UI.

---

## тЬж Features

### ЁЯФР Authentication
- JWT-based login & registration with PBKDF2 password hashing
- Persistent sessions via `localStorage`
- Role-based access control тАФ **Admin**, **Recruiter**, **HiringManager**
- Auto-logout on token expiry (401)

### ЁЯУЛ Dashboard
- **Flip-counter metrics** тАФ Active Roles, Sourced Talent, Pending Offers, Hired This Month (real date-range)
- **Fanning index card cabinet** тАФ hover to fan out the latest candidates
- **Physical hiring sieve funnel** тАФ live percentages across all pipeline stages
- **Desk day-planner** тАФ real interview dates from candidate data, current week auto-calculated
- **Activity audit log** тАФ timestamped feed of all recruiter actions
- **AI Co-Pilot** тАФ Gemini-powered strategic hiring insight widget

### ЁЯСе Candidates
- Full CRUD тАФ add, edit, delete, duplicate candidates
- **Dual view** тАФ card catalog (index cards) + ledger table rows
- Search by name, email, or skills
- Filter by pipeline status, experience level, and favorites
- **Inline form validation** тАФ required fields, email format, phone format, interview date guard
- AI screening against any job spec (Gemini or mock fallback)
- CSV export of the current filtered list
- **RBAC gating** тАФ edit/delete restricted to the owning recruiter or Admins

### ЁЯТ╝ Job Roles
- Full CRUD for job specifications
- **Search & filter** тАФ by title, department, skills, manager; filter by status and type
- **Physical rubber-stamp** status badges (Active / Draft / Closed)
- **Inline form validation** тАФ title, department, description, vacancies
- Admin-only publish, edit, and delete controls

### тЪЩя╕П Settings
- Update name, department, avatar URL
- Change password with server-side PBKDF2 re-hash
- Employee badge display card

### ЁЯПа Landing Page
- Hero, feature highlights, stats strip, FAQ accordion, CTA section

## тЬж API Endpoints

```
POST   /api/auth/register       Register a new user
POST   /api/auth/login          Login and receive JWT
GET    /api/auth/me             Get current user profile
PUT    /api/auth/settings       Update profile / password

GET    /api/jobs                List all job roles
POST   /api/jobs                Create job role (auth required)
PUT    /api/jobs/:id            Update job role (auth required)
DELETE /api/jobs/:id            Delete job role (auth required)

GET    /api/candidates          List all candidates
POST   /api/candidates          Add candidate (auth required)
PUT    /api/candidates/:id      Update candidate (auth required)
DELETE /api/candidates/:id      Delete candidate (auth required)
POST   /api/candidates/:id/favorite   Toggle favorite status

GET    /api/logs                Recent activity logs
POST   /api/ai/screen           AI candidate screening (Gemini)
POST   /api/ai/insights         AI pipeline insights (Gemini)
